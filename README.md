# Deesa UCAT AI Platform

Production-oriented Laravel architecture for an AI-powered UCAT preparation platform with:

- Student and admin roles
- Sequential timed UCAT mock exams
- Deep performance analytics
- Live exam monitoring
- Gemini-powered AI question generation

## Stack

- Laravel 11
- Livewire 3
- Tailwind CSS
- MySQL
- Gemini API via Guzzle

## Key Features

- Apple-inspired glass landing page
- Student dashboard for exam discovery, results, and trends
- Admin dashboard with live monitoring and student analytics
- Timed multi-section exam flow with autosave and section locking
- AI insights for weak areas and practice recommendations
- Question generation pipeline using Gemini

## Setup

This workspace was scaffolded manually because PHP and Composer were not available in the current execution environment. To run it on a PHP-enabled machine:

1. Install PHP 8.3+, Composer, Node.js 20+, and MySQL.
2. Create a new Laravel 11 app in this directory or use this codebase directly.
3. Run `composer install`.
4. Run `npm install`.
5. Copy `.env.example` to `.env` and configure:
   - Database credentials
   - `GEMINI_API_KEY`
   - `GEMINI_MODEL`
6. Run `php artisan key:generate`.
7. Run `php artisan migrate --seed`.
8. Install Breeze with Livewire stack if not already installed:
   - `php artisan breeze:install livewire`
9. Run `npm run build` or `npm run dev`.
10. Start the app with `php artisan serve`.

## Docker Setup

If you want a portable setup that runs the same way on any system with Docker Desktop or Docker Engine installed, use Docker Compose.

### Prerequisites

- Docker
- Docker Compose

### First run

1. Copy `.env.example` to `.env`.
2. Set at minimum:
   - `APP_NAME`
   - `APP_URL=http://localhost:8080`
   - `DB_DATABASE=ucat_platform`
   - `DB_USERNAME=ucat_user`
   - `DB_PASSWORD=secret`
   - `DB_ROOT_PASSWORD=root`
   - `GEMINI_API_KEY=your_key`
3. Start the stack:
   - `docker compose up --build`
4. Open:
   - `http://localhost:8080`

### Services

- `app`: PHP 8.3 FPM container with Composer and Node available
- `web`: Nginx reverse proxy
- `db`: MySQL 8.4
- `node`: asset build container

### Notes

- The entrypoint automatically runs `composer install`, waits for MySQL, then attempts `php artisan migrate --force` and `php artisan db:seed --force`.
- If you mount a complete Laravel app into this repo, the containers will run it without requiring PHP or Composer on the host.
- The current repository still needs the standard Laravel runtime files such as `artisan`, `public/index.php`, and the full framework bootstrap generated by `composer create-project` before it can actually boot the application. Docker solves host portability, but it does not replace the missing Laravel scaffold.

## Architecture

- `app/Http/Controllers`: page and flow orchestration
- `app/Livewire`: interactive dashboards and exam runner
- `app/Models`: exam, analytics, and audit entities
- `app/Services`: AI integration, exam orchestration, analytics
- `database/migrations`: schema
- `database/seeders`: demo data

## Important Notes

- Authentication scaffolding is designed to sit on top of Laravel Breeze or Jetstream.
- Live monitoring assumes periodic Livewire polling.
- Analytics are computed server-side via dedicated service classes.
- The Gemini integration expects strict JSON output and validates the shape before persistence.
